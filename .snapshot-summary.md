# Snapshot Release Summary

**Version**: 2.4.46_develop  
**Comparison**: Changes since **v2.3.0018** (September 13, 2025)  
**Status**: Pre-release / Development Build

---

## ✨ New Features

- **Connection Loss Detection** — Backend unreachability indicator with 10-second timeout, auto-reconnect, and recovery notifications
- **Dashboard Startup Loading Screen** — Professional full-screen loading indicator with animated spinner shown until gateway connection established; eliminates blank/flashing UI and prevents premature overlay display
- **Smart Settings Application Feedback** — Intelligent overlay with context-aware messaging for WiFi/MQTT/DTU setting changes; auto-detects restart vs immediate-apply based on setting type; extended 75-second polling for slow WiFi reconnection
- **Password Field Security** — Passwords masked as dots after page reload with show/hide toggle; works for WiFi and MQTT authentication fields
- **Firmware Update Progress UI** — Real-time visual feedback with animated progress bar, 60-second timeout protection, and automatic page reload after completion
- **Battery & Solar+Battery Remote Monitoring** — New distributed TFT display modes with SOC gauges and stored energy indicators; supports mixed solar+battery systems with dual monitoring flags
- **Alpine.js v2 Dashboard** — New offline-capable web UI at `/index2.html` with solar/battery HUD overlay, modal status badges, and embedded assets (no external CDN dependencies)
- **Three TFT Display Render Methods** — Dedicated rendering for Solar-only, Battery-only, and Combined solar+battery monitoring modes with dynamic text spacing

## 🔧 Improvements

- **WiFi Tab Smart Scanning** — Silent background network refresh when drawer opens with cached networks; visible "Scanning..." only when user explicitly clicks Scan button
- **Dashboard Consolidation** — Removed dual-dashboard architecture; Alpine.js now primary UI at `/index.html`; legacy jQuery dashboard eliminated for cleaner codebase
- **Device Reconnection Handling** — Intelligent polling with 75-second timeout accounts for slow WiFi reconnection after device restart
- **Firmware Upload Now Non-Blocking** — Progress polling starts immediately instead of waiting for full file transfer to complete
- **Dashboard Faster & Offline** — Removed jQuery (~90 KB) and Font Awesome CDN dependency; all assets embedded in firmware
- **Fixed `/doupdate` Endpoint** — Now returns HTTP 200 OK instead of 501 error for manual firmware updates
- **Visual Feedback for Disabled Controls** — Greyed-out buttons with tooltips; version checking shows "disabled" instead of misleading "checking" status
- **Alpine.js UI Firmware Capability Detection** — Automatically enforces display mode constraints based on device capabilities (mutual-exclusive: Solar XOR Battery XOR Both)
- **Enhanced Configuration** — Split `remoteDisplay` flag into `remoteDisplay_SolarMonitor` and `remoteDisplay_BatteryMonitor` for more flexible mode selection

## 🐛 Bug Fixes

- Modal message now shows actual firmware filename during manual update ("Updating with filename.bin") instead of generic "checking"
- Filename automatically cleared after update completes, fails, or times out — prevents stale display
- Duplicate percentage display below progress bar — removed
- Modal no longer reopens after page reload following successful update
- Browser accessibility warnings for password input fields — added proper form context
